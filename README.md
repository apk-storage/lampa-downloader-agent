# Lampa Downloader

**Отправляйте торренты из Lampa прямо на свой компьютер.**

Выберите раздачу на телевизоре или телефоне, нажмите **«Скачать на ПК»** — агент получит команду и скачает файл во встроенном торрент-клиенте.

ПК и устройство с Lampa могут находиться в разных сетях. Проброс портов и отдельный торрент-клиент не требуются.

</div>

---

## Что умеет

- отправлять magnet-ссылки из Lampa на компьютер;
- скачивать торренты встроенным движком;
- показывать прогресс в Lampa и в панели агента;
- автоматически раскладывать фильмы и сериалы по папкам;
- продолжать незавершённые загрузки после перезапуска;
- отменять загрузки;
- работать через NAT без проброса портов;
- шифровать команды между плагином и агентом;
- работать в фоне и запускаться вместе с Windows.

> Агент **скачивает**, а не транслирует видео. Для начала загрузки компьютер должен быть включён, подключён к интернету, а агент — запущен.

---

## Как это работает

```text
Lampa                         relay                         компьютер
┌──────────────────┐         ┌──────────────────┐          ┌──────────────────┐
│ Плагин выбирает  │  WSS    │ Передаёт         │   WSS    │ Агент получает   │
│ magnet-ссылку    │ ──────► │ зашифрованную    │ ──────►  │ команду и качает │
│ и шифрует её     │         │ команду          │          │ торрент          │
└──────────────────┘         └──────────────────┘          └──────────────────┘
                                                                  │
                                                                  ▼
                                                        BitTorrent-раздача
```

Magnet-ссылка, название и категория шифруются на устройстве с Lampa и расшифровываются только агентом. Сам торрент-файл передаётся напрямую между компьютером и участниками BitTorrent-раздачи — не через relay.

# Установка

## 1. Установите плагин в Lampa

Откройте:

**Настройки → Расширения → Добавить плагин**

и укажите адрес:

```text
https://apk-storage.github.io/lampa-plugins/plugin.js
```

После установки в настройках появится раздел **Lampa Downloader**.

---

## 2. Скачайте агент

| Система | Файл |
|---|---|
| **Windows x64** | [lampa-agent.exe](https://github.com/apk-storage/lampa-downloader-agent/releases/latest/download/lampa-agent.exe) |
| **Windows x64, версия с консолью и логами** | [lampa-agent-console.exe](https://github.com/apk-storage/lampa-downloader-agent/releases/latest/download/lampa-agent-console.exe) |
| **macOS Intel** | [lampa-agent-macos-intel](https://github.com/apk-storage/lampa-downloader-agent/releases/latest/download/lampa-agent-macos-intel) |
| **macOS Apple Silicon** | [lampa-agent-macos-arm](https://github.com/apk-storage/lampa-downloader-agent/releases/latest/download/lampa-agent-macos-arm) |
| **Linux x64** | [lampa-agent-linux](https://github.com/apk-storage/lampa-downloader-agent/releases/latest/download/lampa-agent-linux) |

### Windows

Запустите `lampa-agent.exe`.

Если Windows SmartScreen покажет предупреждение:

1. нажмите **Подробнее**;
2. выберите **Выполнить в любом случае**.

Используйте этот вариант только для файла, скачанного из официального раздела Releases данного репозитория.

На Windows агент:

- открывает отдельное окно управления;
- остаётся работать в системном трее после закрытия окна;
- автоматически добавляет себя в автозагрузку текущего пользователя.

Для диагностики можно запустить `lampa-agent-console.exe`: функциональность та же, но дополнительно отображаются технические логи.

### macOS

После скачивания разрешите запуск файла:

```bash
chmod +x lampa-agent-macos-arm
./lampa-agent-macos-arm
```

Для Mac с процессором Intel используйте `lampa-agent-macos-intel`.

Если macOS блокирует неподписанный файл, откройте:

**Системные настройки → Конфиденциальность и безопасность → Всё равно открыть**

Панель управления откроется в браузере. Системного трея и автоматической настройки автозапуска на macOS пока нет.

### Linux

```bash
chmod +x lampa-agent-linux
./lampa-agent-linux
```

Панель управления откроется в браузере. Автозапуск на Linux необходимо настраивать отдельно.

---

## 3. Свяжите агент с Lampa

После запуска агент покажет шестизначный код.

В Lampa откройте:

**Настройки → Lampa Downloader → Подключить ПК**

Введите код из панели агента.

Код привязан к ключу агента и сохраняется между перезапусками. Не передавайте его посторонним: устройство, прошедшее сопряжение, сможет отправлять агенту задания на загрузку.

---

## 4. Отправьте торрент на компьютер

1. Откройте фильм или сериал в Lampa.
2. Перейдите в раздел **Торренты**.
3. Сделайте долгое нажатие на нужную раздачу.
4. Выберите **Скачать на ПК**.

Прогресс отображается:

- в панели агента;
- в Lampa: **Настройки → Lampa Downloader → Загрузки**.

# Папка загрузок

По умолчанию агент сохраняет данные в папку:

```text
Downloads/Lampa
```

Внутри автоматически создаются категории:

```text
Downloads/Lampa/Movies
Downloads/Lampa/Shows
Downloads/Lampa/Other
```

- `Movies` — фильмы;
- `Shows` — сериалы;
- `Other` — задания с неизвестной категорией.

Корневую папку можно изменить в панели агента. Изменение действует для новых заданий и не переносит уже скачанные или активные загрузки.

# Работа в фоне

## Windows

Закрытие окна крестиком **не останавливает агент** — он продолжает работать в трее.

Чтобы снова открыть панель:

1. найдите значок Lampa Downloader в системном трее;
2. нажмите **Открыть панель**.

Чтобы полностью остановить агент, выберите **Выход** в трее или нажмите кнопку **Выход** в панели.

## macOS и Linux

Панель можно закрыть в браузере, но процесс агента продолжит работать в терминале. Для остановки нажмите `Ctrl+C` в терминале или завершите процесс средствами системы.

Локальная панель доступна по адресу:

```text
http://127.0.0.1:47801
```

Она слушает только localhost и не публикуется в локальную сеть.

# Загрузки и раздача

По умолчанию после полного скачивания агент прекращает раздачу торрента, а файлы остаются на диске.

Незавершённые задания сохраняются в конфигурации. После перезапуска агент заново проверяет уже загруженные части и продолжает скачивание.

При отмене задания агент:

- останавливает торрент;
- удаляет его из списка возобновления;
- **не удаляет** уже записанные на диск файлы.

Ненужные частично скачанные файлы необходимо удалить вручную.

# Решение проблем

## Агент показывает «нет связи»

Проверьте:

- есть ли доступ в интернет;
- не блокирует ли антивирус или корпоративный firewall исходящие WebSocket-соединения;
- доступен ли адрес `relay.bwa.ad`;
- не запущена ли устаревшая копия агента.

Перезапустите агент. Повторное сопряжение обычно не требуется.

## Торрент долго находится в состоянии «подключение»

Чаще всего агент не может получить метаданные magnet-ссылки. Возможные причины:

- у раздачи нет доступных участников;
- DHT/UDP заблокированы сетью или VPN;
- firewall блокирует сетевую активность агента;
- сама magnet-ссылка недоступна.

Проверьте другую раздачу с большим числом сидов и разрешите агенту сетевой доступ в firewall.

## Панель не открылась

Откройте вручную:

```text
http://127.0.0.1:47801
```

Если адрес недоступен, убедитесь, что процесс агента запущен. При повторном запуске в Windows вторая копия не создаётся — вместо этого открывается панель уже работающего экземпляра.

## Нужно полностью сбросить сопряжение

1. Полностью закройте агент.
2. Найдите файл `agent.json`.
3. Сохраните резервную копию, если есть незавершённые загрузки.
4. Удалите файл и запустите агент снова.

Будут созданы новые ключи и новый код сопряжения. Настройки папки, список доверенных устройств и данные для возобновления загрузок также сбросятся.

# Конфигурация

Конфигурация хранится в `agent.json`.

Типичные расположения:

| Система | Путь |
|---|---|
| Windows | `%AppData%\lampa-downloader\agent.json` |
| macOS | `~/Library/Application Support/lampa-downloader/agent.json` |
| Linux | `~/.config/lampa-downloader/agent.json` или `$XDG_CONFIG_HOME/lampa-downloader/agent.json` |

Файл содержит:

- приватный и публичный ключи агента;
- выбранную папку загрузок;
- список сопряжённых устройств;
- незавершённые задания;
- настройку раздачи после завершения.

Не публикуйте `agent.json` и не передавайте его другим людям: в нём находится приватный ключ агента.

---

# Параметры командной строки

```text
--relay URL       адрес WebSocket relay
--config PATH     путь к agent.json
--dir PATH        корневая папка загрузок
--seed            продолжать раздачу после завершения
--selftest VALUE  проверить торрент-движок без Lampa и relay
```

Пример:

```bash
lampa-agent-console.exe --dir "D:\Media" --seed
```

Проверка торрент-движка напрямую:

```bash
lampa-agent-console.exe --selftest "magnet:?xt=urn:btih:..."
```

В `--selftest` также можно передать прямую HTTP(S)-ссылку на `.torrent`-файл.

> Параметр `--seed` сохраняет включённое состояние в конфигурации. Чтобы снова отключить раздачу после завершения, закройте агент и установите `"keep_seeding": false` в `agent.json`.

# Автозапуск и удаление

## Windows

При каждом обычном запуске агент регистрирует себя в автозагрузке текущего пользователя:

```text
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```

Имя записи:

```text
LampaDownloader
```

Чтобы удалить агент:

1. завершите его через меню трея;
2. отключите `LampaDownloader` в **Диспетчер задач → Автозагрузка приложений**  
   или удалите запись командой:

```bat
reg delete "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v LampaDownloader /f
```

3. удалите исполняемый файл;
4. при необходимости удалите папку конфигурации `%AppData%\lampa-downloader`.

Скачанные фильмы и сериалы автоматически не удаляются.

## macOS и Linux

Удалите исполняемый файл и, при необходимости, каталог конфигурации. Если вы самостоятельно создавали LaunchAgent, systemd unit или другую запись автозапуска, удалите её отдельно.

# Безопасность и приватность

Для обмена заданиями используется `NaCl box` на базе X25519.

Relay получает технические данные, необходимые для соединения и маршрутизации, например:

- IP-адрес подключившегося клиента;
- время подключения;
- публичные ключи и идентификаторы сопряжения;
- идентификаторы заданий, состояние и процент выполнения.

При этом magnet-ссылка, название и категория передаются в зашифрованном виде и расшифровываются агентом.

Важно:

- relay не передаёт сам торрент-контент;
- торрент скачивается напрямую из BitTorrent-сети;
- ваш IP-адрес может быть виден другим участникам торрент-раздачи;
- шифрование команд не делает использование BitTorrent анонимным;
- используйте VPN, если он необходим в вашей ситуации и разрешён законодательством вашей страны.

# Сборка из исходников

Требуется **Go 1.24 или новее**.

```bash
git clone https://github.com/apk-storage/lampa-downloader-agent.git
cd lampa-downloader-agent
go mod tidy
```

## Windows

Windows-версия использует нативный WebView и системный трей, поэтому для локальной сборки нужен CGO и совместимый C-компилятор. Для запуска также требуется Microsoft Edge WebView2 Runtime.

Фоновая версия:

```bash
go build -ldflags="-H=windowsgui" -o lampa-agent.exe .
```

Версия с консолью:

```bash
go build -o lampa-agent-console.exe .
```

## macOS

Intel:

```bash
CGO_ENABLED=0 GOOS=darwin GOARCH=amd64 go build -o lampa-agent-macos-intel .
```

Apple Silicon:

```bash
CGO_ENABLED=0 GOOS=darwin GOARCH=arm64 go build -o lampa-agent-macos-arm .
```

## Linux x64

```bash
CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -o lampa-agent-linux .
```

Готовые файлы автоматически публикуются в релизе `latest` после изменений исходного кода или workflow.

# Ограничения

- На macOS и Linux пока нет системного трея и встроенной настройки автозапуска.
- Панель управления доступна только локально на `127.0.0.1`.
- Агент принимает magnet-ссылки; обычные `.torrent`-ссылки поддерживаются только режимом `--selftest`.
- В интерфейсе пока нет отдельного управления списком доверенных устройств.
- Отмена загрузки не удаляет частично скачанные файлы.
- Готовые бинарные файлы не подписаны коммерческим сертификатом.
